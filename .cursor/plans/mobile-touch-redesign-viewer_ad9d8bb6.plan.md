# План: мобильный постраничный режим + свайпы

## Цели

- **Смартфоны (портрет)**: UI без мелких целей касания и без лишних отступов.
- **Постраничность**: на экране **1–3 предложения** (segment) — столько, сколько **полностью помещается по высоте**; только полные предложения, без обрезки и без вертикального скролла.
- **Крупный шрифт**: размер текста **не менее 16px**, на практике **18–20px** на мобильных для комфортного чтения; межстрочный интервал увеличен (например, line-height 1.8–2).
- **text-wrap: balance**: для контейнеров предложений использовать `text-wrap: balance`, чтобы строки внутри предложения распределялись равномерно (прогрессивное улучшение; при отсутствии поддержки — обычный перенос).
- **Навигация**: **свайп влево/вправо** перелистывает страницы.
- **Синхронность**: Source/Target всегда показывают один и тот же набор предложений (одну и ту же «страницу») и перелистываются вместе.
- **Без вертикального скролла**: ни в `sourcePanel`, ни в `targetPanel`, ни внутри страницы.

## Где менять

- Верстка и стили: `index.html` (встроенный CSS + разметка).
- Логика рендера/событий: `index.html` (встроенный JS) и/или `TextViewer.js` (если решим сделать его источником правды).

## Архитектура постраничного рендера

- **Источник данных**: `data.segments` (каждый segment = одно предложение).
- **Страница**: группа из **1–3 последовательных segment**, которая **полностью помещается по высоте экрана** (без вертикального скролла). Группировка: идти по segments подряд, добавлять в текущую страницу, пока суммарная высота контента не превысит доступную высоту; каждое предложение — целиком или не включается.
- **Состояние**:
  - `currentPageIndex` (индекс страницы в массиве заранее посчитанных страниц) или эквивалент (например, индекс первого segment текущей страницы).
- **Рендер**:
  - `renderPage(pageIndex)` очищает панели и рендерит **все segment’ы текущей страницы**:
    - `sourcePanel`: конкатенация `segments[i].source` для всех i в этой странице (каждый segment — в своём блоке/параграфе).
    - `targetPanel`: то же для `segments[i].target`.
  - Для контейнеров предложений (например, `.segment`) задать **`text-wrap: balance`** и **шрифт ≥16px** (рекомендуется 18–20px на мобильных).
  - После рендера навешиваются обработчики на `.lexeme` (тап → `mapper.highlightAlignment(id)`).

## Свайпы (touch)

- Слушаем `touchstart/touchmove/touchend` на контейнере текста (например, `.viewer-content`).
- Распознаём горизонтальный свайп:
  - \(|dx| > thresholdX\) (например 60px)
  - и \(|dy| < thresholdY\) (например 30px)
- Действия:
  - свайп влево → следующая страница (например `goToPage(currentPageIndex + 1)`)
  - свайп вправо → предыдущая страница
- `goToPage(i)`:
  - clamp по границам (0 … количество страниц − 1)
  - вызывает `renderPage(i)`
  - (опционально) даёт лёгкую анимацию смены страницы

## Убираем вертикальный скролл

- Для `.text-panel`:
  - убрать `overflow-y: auto`
  - убрать/не использовать `max-height: 700px` на мобильном
- Для мобильного `viewer-content`/панелей:
  - задать высоту под экран и учесть нижнюю панель управления (если она фиксированная)
  - добавить `padding-bottom`, чтобы контент не перекрывался контрол-баром

## Шрифт и текст (мобильный)

- **Размер**: для `.segment` (и при необходимости для лексем) на мобильных — **минимум 16px**, рекомендуется **18–20px**.
- **Межстрочный интервал**: `line-height: 1.8`–`2` для удобочитаемости.
- **text-wrap: balance**: для параграфов/блоков предложений задать `text-wrap: balance`, чтобы строки внутри предложения балансировались; учитывать, что поддержка не везде есть — использовать как прогрессивное улучшение.

## Синхронизация с TTS

- При начале чтения/переходе к сегменту `i`:
  - показать страницу, которая содержит этот segment (вычислить `pageIndex` по `i`, вызвать `goToPage(pageIndex)` перед подсветкой слов).
- Подсветка остаётся прежней и работает в пределах текущей страницы (нескольких предложений).

## Мобильные контролы

- На мобильном вынести TTS-кнопки в **фиксированный нижний контрол-бар**:
  - крупные кнопки, тач-таргеты ≥44px
  - Play/Pause, Settings
- Панель настроек (`#ttsSettingsPanel`) показывать поверх контента (карточкой), чтобы не ломать постраничность.

## Тест-план (минимум)

- Загрузка JSON → показывается первая страница (1–3 предложения по высоте) в обоих панелях.
- Шрифт на мобильном не менее 16px (лучше 18–20px); при поддержке браузером строки внутри предложения балансируются (text-wrap: balance).
- Свайпы переключают страницы; в Source и Target синхронно отображается один и тот же набор предложений.
- Вертикальный скролл отсутствует; на экране только полные предложения.
- Тап по слову подсвечивает выравнивание на обеих панелях в пределах текущей страницы.
- TTS стартует и подсветка слов идёт на текущей странице; при переходе к сегменту на другой странице страница синхронно переключается.
